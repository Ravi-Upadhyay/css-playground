CSS Specificity

Specificity is the means by which a browser decides which CSS property values are the most relevant to an element and therefore will be applied. Specificity is only based on the matching rules which are composed of CSS selectors of different sorts. Says MDN

CSS Terminology

Before diving deep let us get to know some terminology related to CSS and this concept.

  • Selector:
    Selectors will let you select particular element. That can be selected by element selector, class selector, id selector.
  • Pseudo class:
    Pseudo classes used to select special state of element.
  • Pseudo element:
    Pseudo elements are used to style specified parts of elements

How to calculate

  1. We can seperate CSS rules to 4 main categories and each of them adds different value to specificity score.
  2. The correct notation of specificity score takes a form that separates points by coma in an appropriate order. i.e. (0,0,0,0)
  3. Now let's start adding:
    1. Inline Style: Always take precedence. Adds (1,0,0,0) to the score.
    2. ID: Each Id adds (0,1,0,0) to the score.
    3. Class, Pseudo-Class and Attribute: They adds (0,0,1,0) to the score.
    4. Element, Pseudo-Element: They adds (0,0,0,1) to the score.
  4. Few exceptions to keep in mind:
    1. !important can override the inline style.
    2. Selector inside the :not Pseudo-Class gets counted. The Pseudo-Class itself does not gets counted.
    3. Tie! If two selectors have same specificity, Order will come into play.
